# External Integrations

**Analysis Date:** 2025-03-16

## APIs & External Services

**Email/SMTP:**
- Gmail (OAuth2) - Sends notifications via Gmail OAuth2
  - SDK/Client: `googleapis` 161.0.0
  - Auth: Environment-stored client ID/secret (encrypted in DB)
  - Method: `server/services/emailService.ts` - `getGmailAccessToken()`

- Microsoft 365 (OAuth2) - Sends notifications via Outlook/Azure AD
  - SDK/Client: `@azure/msal-node` 3.8.0
  - Auth: Environment-stored client ID/secret/tenant (encrypted in DB)
  - Method: `server/services/emailService.ts` - OAuth2 MSAL flow

- SMTP Direct - Generic SMTP server support
  - SDK/Client: `nodemailer` 7.0.6
  - Auth: Username/password (encrypted in DB)
  - Used for corporate email servers

**CVE/Vulnerability Data:**
- NVD (National Vulnerability Database) - CVE lookups
  - API: `https://services.nvd.nist.gov/rest/json/cves/2.0`
  - Client: Native `fetch` (Node.js 18+)
  - Rate limiting: 6 second delays between requests
  - No authentication required
  - Implementation: `server/services/cveService.ts`

**Console Heartbeat (Central SamurEye Console):**
- Subscription service communicates with central console
  - Endpoint: HTTPS URL stored in database (from appliance settings)
  - Interval: 5 minutes (active), 30 minutes (standby)
  - Payload: Telemetry + subscription status
  - Response: Console commands (whitelist: system_update, restart_service)
  - Retry backoff: 10s, 20s, 40s, 80s
  - Grace period: 72 hours when console unreachable
  - Implementation: `server/services/subscriptionService.ts`

## Data Storage

**Databases:**
- PostgreSQL (primary)
  - Connection: `DATABASE_URL` environment variable
  - Client: `pg` 8.16.3
  - ORM: Drizzle ORM 0.39.1
  - Schema definition: `shared/schema.ts`
  - Migrations: `migrations/` (auto-generated by Drizzle)
  - Session storage: `connect-pg-simple` (passport sessions in `sessions` table)

**File Storage:**
- Local filesystem only - No cloud storage integration
- Logs: Written to appliance filesystem
- Migrations: Stored in `migrations/` directory

**Caching:**
- React Query (client-side HTTP cache)
- In-memory session store for development (`memorystore` 1.6.7)
- PostgreSQL for production sessions (`connect-pg-simple`)
- No distributed cache (Redis not used)

## Authentication & Identity

**Auth Providers:**

1. **Local Database Auth:**
   - Username/password stored in PostgreSQL
   - Implementation: `server/localAuth.ts`
   - Hash: bcryptjs with salt rounds 12
   - Session: Passport serialization → PostgreSQL (connect-pg-simple)
   - Dev bootstrap: `admin@example.com` / `admin` created on first run

2. **Replit Platform Auth (Development):**
   - OpenID Connect integration for Replit deployment
   - Provider: Replit's built-in OIDC server
   - SDK: `openid-client` 6.7.1
   - Strategy: Passport OpenID Connect strategy
   - Implementation: `server/replitAuth.ts`
   - Scope: `openid email profile offline_access`
   - Fallback: Uses local auth if not on Replit

3. **LDAP/Active Directory (Scanning Only):**
   - Not used for authentication
   - Used by AD Security scanner to query domain information
   - Client: `ldapts` 8.0.9
   - Implementation: `server/services/scanners/adScanner.ts`

## Environment Configuration

**Required env vars:**
- `DATABASE_URL` - PostgreSQL connection string (e.g., `postgres://user:pass@host:5432/samureye`)
- `NODE_ENV` - `development` or `production`
- `PORT` - Server port (default: 5000)
- `ALLOWED_ORIGINS` - Comma-separated CORS whitelist (empty = allow all on single host)

**Optional env vars:**
- Email OAuth2 credentials (stored encrypted in database after initial setup)
- Console URL and API key (stored encrypted in database)

**Secrets location:**
- Environment variables (runtime)
- Database encrypted fields:
  - Credential passwords: `credentials.secretEncrypted` (AES-256-GCM)
  - Email OAuth2 tokens: `settings.oauth2RefreshToken` encrypted
  - Data Encryption Key (DEK): Encrypted with Key Encryption Key (KEK)
  - Implementation: `server/services/encryption.ts`
