Create `scripts/calibrate.ts` — a standalone CLI script runnable via `npx tsx scripts/calibrate.ts`.

**DB Connection** (mirror server/db.ts pattern):
```typescript
import { Pool } from 'pg';
import { drizzle } from 'drizzle-orm/node-postgres';
import { eq, isNotNull } from 'drizzle-orm';
import * as schema from '../shared/schema';
import { readFileSync, writeFileSync } from 'fs';
import { resolve } from 'path';
```

Initialize pool with `process.env.DATABASE_URL`. If DATABASE_URL is not set, print `[calibrate] ERROR: DATABASE_URL not set` and exit 1.

**Query**: SELECT threats joined with hosts where contextualScore IS NOT NULL:
```typescript
const rows = await db
  .select({
    severity: schema.threats.severity,
    source: schema.threats.source,
    evidence: schema.threats.evidence,
    contextualScore: schema.threats.contextualScore,
    scoreBreakdown: schema.threats.scoreBreakdown,
    hostType: schema.hosts.type,
  })
  .from(schema.threats)
  .leftJoin(schema.hosts, eq(schema.threats.hostId, schema.hosts.id))
  .where(isNotNull(schema.threats.contextualScore));
```

**Empty DB guard**: If `rows.length === 0`, write report with status `SKIPPED: no scored threats in database`, log to stdout, exit 0 without patching.

**Stdout prefix**: All console output lines must start with `[calibrate] `.

**THRT-06 Severity Weight Validation**:
- Group rows by severity (critical, high, medium, low)
- Compute average `scoreBreakdown.rawScore` per severity group (cast scoreBreakdown to `any`, access `.rawScore`)
- Check strict ordering: avg(critical) > avg(high) > avg(medium) > avg(low)
- If any inversion found, flag as FAIL and compute adjustment needed
- Log: `[calibrate] THRT-06: PASS` or `[calibrate] THRT-06: FAIL — {inversion description}`

**THRT-08 Host Criticality Validation**:
- Group rows by hostType (domain, server/firewall/router, desktop/switch/other)
- For rows within the SAME severity level, compute average rawScore per host-type group
- Check strict hierarchy: domain avg > server-group avg > desktop-group avg
- Log: `[calibrate] THRT-08: PASS` or `[calibrate] THRT-08: FAIL — {inversion description}`

**THRT-09 Exploitability Validation**:
- Split rows into confirmed (source='nmap_vuln' OR evidence.nucleiMatch OR evidence.confirmed) and unconfirmed
- For rows within the SAME severity level, compute average rawScore for each group
- Check ordering: confirmed avg > unconfirmed avg
- Check ratio: confirmed avg / unconfirmed avg should be approximately 1.3 (within 10% tolerance, i.e., between 1.17 and 1.43)
- Log: `[calibrate] THRT-09: PASS` or `[calibrate] THRT-09: FAIL — {description}`

**Auto-patch** (if ANY component fails):
- Read `server/services/scoringEngine.ts` as string
- Log old values: `[calibrate] CHANGED: SEVERITY_WEIGHTS.critical 100 -> {newValue}` (etc.)
- Use regex with dotAll flag (`/s`) to match and replace the specific constant block
- For SEVERITY_WEIGHTS: replace the block matching `/const SEVERITY_WEIGHTS: Record<string, number> = \{[^}]+\}/s`
- For CRITICALITY_MULTIPLIERS: same regex pattern approach
- For exploitability multiplier (hardcoded 1.3 value): adjust the literal in the ternary expression
- Write patched file back
- Re-verify by re-computing scores inline using the new constant values (do NOT re-import scoringEngine.ts due to module caching -- inline the formula with new values)

**Report output**: Write to `.planning/phases/06-calibration-and-quality/CALIBRATION-REPORT.md` with this structure:

```
# Calibration Report -- {ISO date}

## Summary
- THRT-06 (Severity Weights): {PASS|FAIL}
- THRT-08 (Host Criticality): {PASS|FAIL}
- THRT-09 (Exploitability): {PASS|FAIL}

## Data Summary
- Total scored threats analyzed: {count}
- Severity distribution: critical={n}, high={n}, medium={n}, low={n}
- Host type distribution: domain={n}, server={n}, desktop={n}, other={n}

## Component Details

### THRT-06: Severity Weights
{Per-severity average scores, ordering check result}

### THRT-08: Host Criticality Multipliers
{Per-host-type average scores within same severity, ordering check result}

### THRT-09: Exploitability Multiplier
{Confirmed vs unconfirmed average scores, ratio check result}

## Changes Made
{CHANGED lines or "NO CHANGES -- all components passed"}
```

**Cleanup**: Call `await pool.end()` before exiting.

**Error handling**: Wrap main() in try/catch. On DB connection error, log `[calibrate] ERROR: DB connection failed -- {message}` and exit 1.
