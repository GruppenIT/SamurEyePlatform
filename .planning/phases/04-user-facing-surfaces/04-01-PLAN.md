  <action>
Rewrite `threats.tsx` to implement grouped threat display and remediation preview.

**Client-side grouping logic** (in a useMemo):
- Fetch threats from `/api/threats` using existing useQuery pattern.
- Separate into 3 categories: `parents` (groupingKey !== null AND parentThreatId === null), `children` (parentThreatId !== null grouped into a Map by parentThreatId), `standalone` (both null).
- Filter out children from the top-level list — they only appear inside their parent's Collapsible.

**Grouped threat rows:**
- Each parent threat renders as a `Collapsible` (from `@/components/ui/collapsible`).
- `CollapsibleTrigger` shows: severity Badge, threat title, child count badge (e.g., "3 findings"), contextualScore if present, and remediation preview area.
- `CollapsibleContent` renders child threats in an indented sub-list with their own severity badges and titles.
- Standalone threats render as regular rows (same style as parents but no expand trigger).

**Remediation preview on parent groups (UIFN-04):**
- For each visible parent threat, lazy-fetch its recommendation using `useQuery` with `queryKey: ['/api/threats', parentId, 'recommendation']` and `enabled: !!parentId`. Use `staleTime: 60_000` to avoid refetching.
- Display in a compact inline format: first fix step truncated to ~80 chars + effort tag Badge (e.g., "hours", "minutes").
- If no recommendation exists, show "Sem recomendacao" in muted text.
- NOTE: To avoid N+1 queries for many parents, batch by only enabling queries for parents currently visible/expanded. Acceptable for v1: fetch for all parents since the count is typically 5-15 groups.

**Preserve existing features:**
- Keep severity filter tabs/buttons if they exist in current threats.tsx.
- Keep status filter (open/mitigated/closed) if present.
- Apply filters to parent threats; if a filter matches any child, the parent should still show.
- Keep the existing threat detail dialog trigger (clicking a threat title opens detail).

**Evidence label mapping (UIFN-02):**
Create an `EVIDENCE_LABELS` constant mapping evidence JSONB keys to Portuguese display labels:
```
port -> Porta, protocol -> Protocolo, service -> Servico, version -> Versao,
cveId -> CVE, severity -> Severidade, matcherName -> Correspondencia,
extractedResults -> Resultados, host -> Host, ip -> IP, hostname -> Hostname,
os -> Sistema Operacional, product -> Produto, scriptId -> Script,
output -> Saida, state -> Estado, reason -> Razao
```
Render evidence as a `<dl>` grid (2 cols) with `<dt>` for label and `<dd>` for value. Skip null/undefined/empty values. For nested objects, render as formatted text (NOT JSON.stringify). For arrays, render as comma-separated or bulleted list.

**Threat detail dialog (UIFN-01):**
When a threat row is clicked, open the existing Dialog component showing 3 sections:
1. **Problema** (from recommendation.whatIsWrong or threat.description as fallback)
2. **Impacto** (from recommendation.businessImpact or "Impacto nao avaliado" as fallback)
3. **Correcao** (from recommendation.fixSteps rendered as ordered list, with verification step below)

Lazy-fetch recommendation using `useQuery` with `enabled: !!selectedThreatId`.

Below the 3 sections, show the evidence table (using the label mapping above).

Show effort tag and required role as Badges in the dialog header area.

Do NOT use JSON.stringify anywhere in the rendered output for evidence or recommendation data.
