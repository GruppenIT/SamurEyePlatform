**Server endpoint — `GET /api/posture/coverage` in `server/routes/dashboard.ts`:**

Add inside `registerDashboardRoutes()`:

Query: For each of the 4 journey types (attack_surface, ad_security, edr_av, web_application), find the most recent completed job and count open threats in that category.

Implementation using raw SQL (similar to existing patterns in the file):
```sql
SELECT
  j.type as "journeyType",
  MAX(jobs.finished_at) as "lastRunAt",
  (SELECT status FROM jobs WHERE journey_id IN (SELECT id FROM journeys WHERE type = j.type) AND status IN ('completed','failed','timeout') ORDER BY finished_at DESC LIMIT 1) as "lastStatus",
  COALESCE((SELECT COUNT(*)::int FROM threats WHERE category = j.type AND status = 'open' AND parent_threat_id IS NULL), 0) as "openThreatCount"
FROM (VALUES ('attack_surface'), ('ad_security'), ('edr_av'), ('web_application')) AS j(type)
LEFT JOIN journeys ON journeys.type = j.type
LEFT JOIN jobs ON jobs.journey_id = journeys.id AND jobs.status IN ('completed','failed','timeout')
GROUP BY j.type
```

Alternative simpler approach: 4 separate small queries (one per journey type) if the join is complex. Return shape:
```typescript
[
  { journeyType: string, lastRunAt: string | null, lastStatus: 'completed' | 'failed' | 'timeout' | null, openThreatCount: number }
]
```

Use `db.execute(sql`...`)` like the existing patterns in the file. Wrap in try/catch, return 500 on error.
