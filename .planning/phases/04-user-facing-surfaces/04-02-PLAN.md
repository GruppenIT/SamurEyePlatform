**Server endpoint — `GET /api/action-plan` in `server/routes/dashboard.ts`:**

Add a new route inside `registerDashboardRoutes()`:

```
GET /api/action-plan?effortTag=&roleRequired=&category=
```

Implementation:
1. Parse optional query params: `effortTag` (string), `roleRequired` (string), `category` (string — maps to threat category/journey type).
2. Query: SELECT recommendations joined with threats on `recommendations.threatId = threats.id`.
3. Filter: Only include recommendations where the associated threat has `status = 'open'` AND `parentThreatId IS NULL` (only parent/standalone threats, not children).
4. Apply optional filters: `recommendations.effortTag`, `recommendations.roleRequired`, `threats.category`.
5. Order by `threats.contextualScore DESC NULLS LAST` (highest priority first).
6. Return array of objects with shape:
```typescript
{
  recommendationId: string;
  threatId: string;
  threatTitle: string;
  threatSeverity: string;
  threatCategory: string | null;
  contextualScore: number | null;
  projectedScoreAfterFix: number | null;
  whatIsWrong: string;
  fixPreview: string;       // fixSteps[0] or ''
  effortTag: string | null;
  roleRequired: string | null;
  status: string;
}
```

Use Drizzle's `db.select({...}).from(recommendations).innerJoin(threats, eq(recommendations.threatId, threats.id))` pattern. Apply `where()` conditions for status=open, parentThreatId IS NULL, and optional filters. Apply `orderBy(desc(threats.contextualScore))`.

Import `recommendations` and `threats` from `@shared/schema`, `eq`, `and`, `desc`, `isNull` from `drizzle-orm`.

**Client page — `client/src/pages/action-plan.tsx`:**
