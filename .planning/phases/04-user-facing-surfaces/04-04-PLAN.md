Add WebSocket-triggered React Query cache invalidation to `postura.tsx`.

1. Import `useWebSocket` from `@/lib/websocket` (or wherever useWebSocket is exported — check the import used in active-jobs.tsx).
2. Import `queryClient` from `@/lib/queryClient`.
3. Add a `useEffect` that watches `lastMessage`:

```typescript
const { lastMessage } = useWebSocket();

useEffect(() => {
  if (
    lastMessage?.type === 'jobUpdate' &&
    ['completed', 'failed', 'timeout'].includes(lastMessage.data?.status)
  ) {
    // Invalidate all dashboard-relevant query keys
    queryClient.invalidateQueries({ queryKey: ['/api/posture/history'] });
    queryClient.invalidateQueries({ queryKey: ['/api/posture/coverage'] });
    queryClient.invalidateQueries({ queryKey: ['/api/action-plan'] });
    queryClient.invalidateQueries({ queryKey: ['/api/threats'] });
  }
}, [lastMessage]);
```

IMPORTANT: Only invalidate on terminal statuses (`completed`, `failed`, `timeout`). Do NOT invalidate on `running` or `pending` — that would cause excessive refetches (Pitfall 3 from research).

This ensures PostureHero, JourneyCoverage, and TopActions all refresh their data automatically when any job finishes, without the user needing to refresh the page.
