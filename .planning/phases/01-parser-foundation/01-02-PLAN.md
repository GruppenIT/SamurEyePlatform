  </behavior>

       NucleiFindingSchema extends BaseFinding with:
       - `type: z.literal('nuclei')`
       - `templateId: z.string()`
       - `matchedAt: z.string()` (the URL that matched)
       - `matcherName: z.string().optional()` (PARS-06)
       - `extractedResults: z.array(z.string()).optional()` (PARS-06)
       - `curlCommand: z.string().optional()` (PARS-06)
       - `info` object: `{ name, severity, description?, tags?: string[], classification?: { cveId?: string[], cweId?: string[] }, references?: string[], remediation?: string }`
       - `host: z.string().optional()`
       - `port: z.string().optional()`
       - Call `.strip()`
       - Export type: `NucleiFinding = z.infer<typeof NucleiFindingSchema>`

       - `info-severity-with-tags.jsonl`: One line with severity 'info', tags `['tech', 'detect']`, description and references. Tests low-severity handling.
       - `malformed-mixed-lines.jsonl`: Three lines: first valid, second is `{invalid json`, third valid. Tests skip-bad-line behavior.

       NOTE: nuclei JSONL uses kebab-case field names (`matcher-name`, `extracted-results`, `curl-command`, `matched-at`, `template-id`). The parser must map these to camelCase for the Zod schema.

    3. Rewrite `parseNucleiOutput` in `vulnScanner.ts`:
       - Rename to `parseNuclei(stdout: string): NucleiFinding[]`
       - Split stdout by newlines, filter empty lines
       - For each line: `JSON.parse()` in try-catch (log.warn + skip on failure)
       - Map nuclei kebab-case fields to camelCase: `template-id` -> `templateId`, `matched-at` -> `matchedAt`, `matcher-name` -> `matcherName`, `extracted-results` -> `extractedResults`, `curl-command` -> `curlCommand`
       - Map `info.classification.cve-id` -> `cveId`, `info.classification.cwe-id` -> `cweId`
       - `NucleiFindingSchema.safeParse(mapped)` — push valid, log+skip invalid
       - Keep the old `parseNucleiOutput` method as `@deprecated` wrapper calling `parseNuclei`
       - Preserve the existing `VulnerabilityFinding` interface for backward compat

       - Specific assertions: `xss-with-matcher` has matcherName, extractedResults, curlCommand populated
