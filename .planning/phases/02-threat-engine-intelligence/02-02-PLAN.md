  <action>
Add a new method `groupFindings(jobId: string, journeyType: string)` to the ThreatEngineService class. This runs AFTER `runJourneyPostProcessing()` in `processJobResults()`.

**groupFindings() logic:**

1. Query all threats for this jobId that have no parentThreatId (i.e., ungrouped child threats from this job).

2. Compute grouping key for each child based on journeyType:
   - `attack_surface`: Check if evidence has a CVE. If yes → `grp:as:cve:{cveId}`. If no → `grp:as:{normalizeHost(evidence.host)}:{classifyServiceCategory(evidence.port, evidence.service).category}`
   - `ad_security`: `grp:ad:{evidence.adCheckCategory || 'general'}:{evidence.domain || 'unknown'}`
   - `edr_av`: `grp:edr:{hostId}` (use the threat's hostId directly)
   - `web_application`: `grp:wa:{normalizeHost(evidence.host)}:{evidence.templateTags?.[0] || evidence.type || 'general'}`

3. Group children by computed grouping key (Map<string, Threat[]>).

4. For each group with 1+ children:
   a. Derive parent title from grouping key (e.g., "Exposed Administration Services on 192.168.1.10" for admin category).
   b. Derive parent severity = highest among children.
   c. Derive parent status = open if any child active, else mitigated.
   d. Call `upsertParentThreat()` with groupingKey, derived title/severity/status, same journeyId/jobId/hostId/assetId/category as children.
   e. For each child in group, call `linkChildToParent(child.id, parent.id)`.

5. Log group counts at info level.

**Parent title derivation** (use SERVICE_CATEGORIES label for attack_surface):
- Attack Surface service: `"Exposed {SERVICE_CATEGORIES[cat].label} on {host}"`
- Attack Surface CVE: `"CVE-{cveId} Across Multiple Hosts"`
- AD Security: `"{adCheckCategory} Issues in {domain}"`
- EDR/AV: `"Endpoint Security Issues on {hostname}"`
- Web Application: `"Web Vulnerabilities ({tag}) on {host}"`

Use the existing `normalizeHost()` helper inside `computeCorrelationKey` — extract it to a class-level or module-level helper so both methods can use it. Currently it is a closure inside computeCorrelationKey.
