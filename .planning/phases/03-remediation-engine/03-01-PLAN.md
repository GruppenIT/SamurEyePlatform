1. **RecommendationEngine** — Create `server/services/recommendationEngine.ts` as a class-based singleton (matching scoringEngine pattern):
   - `async generateForJob(jobId: string): Promise<void>`:
     a. Query threats for jobId where `parentThreatId IS NULL` (gets both parent groups and standalone threats — NOT children)
     b. For each threat:
        - Resolve host via `threat.hostId` if present (use storage.getHost or equivalent)
        - Determine `hostFamily` from host enrichment data (default 'other' if no host)
        - If threat has children (`groupingKey IS NOT NULL`): fetch via `getChildThreats(threat.id)`, aggregate child evidences
        - If standalone (no groupingKey, no parentThreatId): use `threat.evidence` directly
        - Look up template via `getTemplate(threat.ruleId || threat.category)` from the dispatch map
        - If no template found: log warning, skip (don't crash pipeline)
        - Call `template.generate(ctx)` to get `GeneratedRecommendation`
        - Call `upsertRecommendation()` with generated data + threatId + templateId
     c. Log summary: `{ jobId, generated: N, skipped: M }`
   - `async syncRecommendationStatus(threatId: string, newThreatStatus: string): Promise<void>`:
     a. Look up recommendation by threatId
     b. If no recommendation exists, return silently
     c. Status transitions: 'mitigated' -> recommendation.status = 'applied'; 'open' (reactivated) -> 'failed'; 'closed' -> 'verified'
     d. Update recommendation status + updatedAt
   - Export singleton: `export const recommendationEngine = new RecommendationEngine();`
   - Create logger: `const log = createLogger('recommendationEngine');`
